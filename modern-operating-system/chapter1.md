# 第一章 序論

## 1.1 オペレーティングシステムとは何か

1. ハードウェアを隠し、アプリケーションプログラムに対して美しい抽象化を提供するもの (トップダウンな視点)
2. 複雑なシステムの構成要素全てを管理するもの (ボトムアップな視点)

## 1.2 OSの歴史

- 第一世代：真空管 機械語による数値計算が主
- 第二世代：トランジスタ 信頼性と速度が向上、メインフレームのバッチシステム上で複数のジョブが動くように OSのはしり
- 第三世代：ICとマルチプログラミング IBMが開発したOS/360が代表的だが、コードベースは肥大化しバグも多いが、商用分野で入出力の間にCPU時間が無駄にならないように新しいパラダイムを導入した
  - マルチプログラミング: メモリ上で3つのジョブが並行して行われる
  - これはタイムシェアリングへと発展していく
